# Mono GameMaker - Project Orchestrator

Mono GameMaker is a lightweight, pragmatically designed C# desktop IDE built specifically as a Code/AI-First orchestrator for MonoGame projects. Rejecting traditional UI-First heavy click workflows, the IDE functions as a spatial arrangement canvas and JSON/Prefab registry, delegating 100% of entity behavior to clean, compile-ready C# code generated by developers or autonomous AI agents.

## Key Features

1. **Dynamic Binary Hot Reload**: Uses .NET 8 `AssemblyLoadContext` to load compiled user assemblies in memory from byte streams. This achieves a collectible, hot-swappable DLL cycle without locking physical files on disk, ensuring subsequent background compiles never crash due to write locks.
2. **Live Tweaking via Reflection**: Renders and binds public variables, fields, and properties of simulated script instances in real-time inside the ImGui Inspector. Modifications made during simulation are applied instantly in memory for rapid gameplay tuning.
3. **In-Game Telemetry Pipeline**: Intercepts stdout/stderr streams to capture target executable outputs, warnings, and compilation logs dynamically, redirecting telemetry to the IDE console panel for active error handling.
4. **Shared Runtime Architecture**: Employs a single source of truth where the IDE editor viewport simulation and the scaffolded game project compile against the exact same runtime contracts (`GameEntity` and `IEntityScript`), guaranteeing identical behavior.
5. **Immediate Mode UI**: Built on top of MonoGame using `ImGui.NET`, providing high-performance widgets with low boilerplate and zero MVVM/XAML overhead.
6. **Reactive Filesystem Cache**: Monitored by a thread-safe `FileSystemWatcher` featuring a 150ms debounce window. Filesystem changes are scanned in the background and safely swapped using reference locking to prevent UI frame drops and collection modification exceptions.
7. **Native Window Splits & Layouts**: Uses custom P/Invoke bindings directly to `cimgui.dll` to perform programmatic window splits. You can toggle layouts from the **VIEW** menu:
   - **Default Layout**: Toolbar (top), Project Explorer (left), Inspector (right), and Console Output (bottom).
   - **Wide Console Layout**: Expands the logs window to 60% of the center workspace.
   - **Focused Editor Layout**: Minimizes panels to maximize canvas workspace.

## How to Run

To run the IDE in development mode, execute:

```bash
dotnet run --project src/IDE/IDE.csproj
```

Alternatively, you can open the published Windows executable directly:
`src/IDE/bin/Release/net8.0/win-x64/publish/IDE.exe`
